مشروع متجر Shrkawy Store التجريبي

نص المطلوب: إنشاء نموذج متجر إلكتروني متكامل لبيع الإلكترونيات، يعمل من التصفح والسلة وCheckout حتى الدفع التجريبي وصفحات النتيجة، مع ملفات المشروع وREADME وتوثيق PDF.

فكرة المتجر

Shrkawy Store متجر إلكتروني تعليمي لبيع الإلكترونيات والإكسسوارات التقنية. يهدف إلى توضيح دورة الشراء الكاملة، مع تصميم عربي RTL وسلة مستمرة ومحاكي دفع لا ينفذ خصمًا حقيقيًا.

خطوات إنشاء متجر إلكتروني

1. تحديد فكرة المتجر والمنتجات

تحديد التخصص والقيمة المقدمة، ثم اختيار منتجات واضحة الطلب والهوامش والمخاطر. في هذا المشروع يركز المتجر على الإلكترونيات والإكسسوارات التقنية.

2. تحديد العملاء المستهدفين

وصف الشرائح العمرية والقدرة الشرائية واحتياجات الاستخدام لتوجيه الأسعار والتصميم والحملات.

3. اختيار الاسم والهوية البصرية

اختيار اسم سهل التذكر وألوان وخطوط وشعار متسق. تم استخدام Shrkawy Store وهوية تقنية داكنة مع البنفسجي والأزرق.

4. اختيار النطاق

حجز اسم نطاق قصير وقريب من العلامة التجارية مع امتداد مناسب مثل .com أو .store.

5. اختيار الاستضافة

اختيار استضافة حسب الزيارات والأمان والحاجة إلى Backend؛ الواجهة الساكنة تناسب Netlify أو Vercel.

6. تصميم الصفحات وتجربة المستخدم

بناء مسار واضح: الرئيسية، المنتجات، التفاصيل، السلة، Checkout، الدفع، والنتيجة، مع دعم الهاتف وRTL.

7. إضافة المنتجات والمخزون

إدخال الاسم والوصف والصور والسعر والخصم والمواصفات والكمية المتاحة وتحديثها بصورة منتظمة.

8. إنشاء السلة

حفظ المنتجات والكميات، حساب الإجماليات، وتوفير الزيادة والنقص والحذف. استخدم المشروع localStorage.

9. إنشاء Checkout

جمع بيانات العميل والعنوان، عرض ملخص الطلب، التحقق من الحقول، واختيار وسيلة الدفع.

10. ربط بوابة الدفع

استخدام Hosted Checkout أو خادم آمن ومفاتيح بيئة. النسخة الحالية تستخدم محاكيًا تعليميًا، وتضم خادم Stripe اختياريًا.

11. ربط الشحن

التكامل مع شركة شحن لحساب التكلفة وإنشاء بوليصة وتتبع الطلب، أو تطبيق قواعد شحن ثابتة كبداية.

12. سياسات الشحن والإرجاع

كتابة مدد التوصيل والتكاليف وشروط الإرجاع والاستبدال بوضوح قبل الدفع.

13. تأمين الموقع

تفعيل HTTPS، عدم تخزين بيانات البطاقة، التحقق من المدخلات، حماية الأسرار، وتحديث المكونات.

14. اختبار المتجر

اختبار الروابط والسلة والحسابات والتحقق وحالات الدفع الناجحة والمرفوضة والملغاة وعلى أجهزة متعددة.

15. النشر والتسويق والتحليل

نشر النسخة المستقرة، تحسين محركات البحث، تشغيل الحملات، ومراقبة التحويل ومعدل التخلي عن السلة.

الصفحات الموجودة

  • الرئيسية والمنتجات.
  • تفاصيل المنتج.
  • سلة المشتريات.
  • Checkout وبيانات العميل.
  • صفحة محاكي الدفع.
  • صفحات النجاح والفشل والإلغاء.

التقنيات المستخدمة

HTML5 وCSS3 وJavaScript وlocalStorage، مع Node.js/Express اختياري لإنشاء Stripe Checkout Session باستخدام متغيرات البيئة.

كيفية عمل السلة

تُخزن العناصر على شكل رقم منتج وكمية داخل localStorage. عند كل تعديل يُعاد حساب إجمالي المنتجات والشحن والخصم والإجمالي النهائي، ويُحدّث عداد السلة تلقائيًا.

كيفية عمل Checkout

يقرأ الصفحة محتويات السلة، تعرض ملخصًا دقيقًا، وتتحقق من الاسم والهاتف والبريد والعنوان. بعد ذلك تُنشئ بيانات طلب تجريبي وتنتقل إلى المحاكي عند اختيار البطاقة.

الدفع الحقيقي والدفع التجريبي

الدفع الحقيقي يتصل بمزود دفع ويجري تفويضًا وتسوية مالية. أما المحاكي فيكرر واجهة وحالات العملية فقط دون اتصال مصرفي أو خصم أموال. أضيف خادم اختياري لربط Stripe Test Mode دون وضع المفتاح السري في الواجهة.

طريقة تجربة الدفع

نجاح: 4242 4242 4242 4242. رفض: 4000 0000 0000 0002. تاريخ 12/30 وCVV 123. ويمكن اختيار نجاح أو رفض أو إلغاء يدويًا.

طريقة التشغيل

من داخل المجلد نفّذ python -m http.server 5500 ثم افتح http://localhost:5500.

رابط المتجر

لا يوجد رابط عام منشور من بيئة التنفيذ الحالية. المشروع جاهز للنشر على Netlify أو GitHub Pages للمحاكي، وخطوات النشر موجودة في README.
